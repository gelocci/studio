# Fluxo — Melhoria de Página

## 1. Objetivo

Este fluxo serve para melhorar uma página do ecossistema Gelocci, especialmente páginas públicas do `www`, como home, ferramentas, páginas educativas e landing pages.

---

## 2. Quando usar

Use este fluxo quando o Gerson quiser:

- melhorar uma página existente;
- criar uma nova página;
- melhorar layout;
- melhorar copy;
- melhorar SEO;
- melhorar conversão;
- melhorar responsividade;
- aproximar uma página do design desejado.

Exemplos:

```text
Melhorar a home
Criar landing do Asset Allocation
Melhorar página de juros compostos
Melhorar página educativa sobre renda fixa
Melhorar página da calculadora IR
```

---

## 3. Agentes participantes

Agentes obrigatórios:

- Orquestrador Gelocci;
- Produto Gelocci;
- UX/UI Gelocci;
- SEO Gelocci;
- Arquiteto Gelocci;
- Desenvolvedor Gelocci;
- QA Gelocci;
- Studio Lead.

Agentes condicionais:

- Financeiro Gelocci, se houver conceito financeiro, fórmula ou afirmação sensível;
- Segurança Gelocci, se houver scripts externos, formulários, cookies ou dados;
- DevOps Gelocci, se houver publicação;
- Revisor Gelocci, após implementação.

---

## 4. Entradas necessárias

O fluxo deve receber:

- página alvo;
- objetivo da melhoria;
- público-alvo;
- problema percebido;
- arquivos atuais, se houver;
- prints ou descrição visual, se houver;
- system design aplicável;
- layout guidelines, se houver;
- restrições técnicas.

---

## 5. Sequência do fluxo

```text
Gerson solicita melhoria
  ↓
Orquestrador classifica a demanda
  ↓
Produto avalia objetivo e valor
  ↓
UX/UI propõe experiência
  ↓
SEO avalia estrutura e intenção de busca
  ↓
Financeiro valida conceitos, se aplicável
  ↓
Arquiteto valida solução técnica
  ↓
Desenvolvedor propõe implementação
  ↓
Orquestrador consolida proposta
  ↓
Studio Lead revisa
  ↓
Gerson aprova
  ↓
Desenvolvedor gera arquivos completos
  ↓
Revisor, QA e Segurança validam
  ↓
DevOps orienta publicação, se aplicável
```

---

## 6. Saída esperada

O fluxo deve gerar:

```text
Página:
Objetivo:
Problema identificado:
Proposta de melhoria:
Estrutura recomendada:
Ajustes UX/UI:
Ajustes SEO:
Ajustes de conteúdo:
Ajustes técnicos:
Arquivos afetados:
Riscos:
Critérios de aceite:
Decisão do Studio Lead:
```

---

## 7. Critérios de aceite

Uma melhoria de página deve:

- deixar a proposta mais clara;
- melhorar experiência;
- preservar performance;
- manter conteúdo indexável;
- funcionar bem em mobile;
- respeitar o layout do produto;
- evitar complexidade desnecessária;
- ter CTA claro quando fizer sentido.

---

## 8. Regras específicas para o www

- Priorizar HTML, CSS e JavaScript puro.
- Não introduzir backend.
- Não usar biblioteca pesada sem justificativa.
- Manter SEO e performance como requisitos.
- Preservar clareza e didática.
- Garantir que texto importante esteja no HTML.

---

## 9. Resultado ideal

Ao final, a página deve ficar mais clara, bonita, útil, rápida e alinhada ao objetivo do ecossistema Gelocci.
